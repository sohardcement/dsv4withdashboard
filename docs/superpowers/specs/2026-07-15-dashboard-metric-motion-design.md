# Dashboard 核心指标方向滚动动效设计

## 背景与目标

监控模式顶部核心指标目前在轮询后直接替换文本，并使用统一的淡入/下移反馈。用户确认采用 B 方案：数字更新时有明显的滚轮感，并根据数值变化方向上下滑动。本次改造只作用于监控模式顶部指标，不改变管理模式、调用表格或容量设置数字。

## 已确认的设计

### 覆盖范围

动效只覆盖 `#monitorMetrics` 中的六个核心指标：

- 当前阶段 / 活动 / 服务：保留文字更新，不对阶段和服务名播放数字动效；
- 预填充：以平均速度作为方向比较值；
- 解码：以平均速度作为方向比较值；
- 请求 KV 命中：以命中率作为方向比较值；
- 上下文利用率：以利用率作为方向比较值；
- 队列 / 客户端：以队列深度作为方向比较值。

首次获得数值、数值不可用或前后值相等时只更新内容，不播放动效。

### 双层滚动

每个可动指标保留一个固定高度的视口和两层值文本：旧值负责离场，新值负责入场。更新时先写入新值，再通过方向 class 触发一次 CSS 动画；动画结束后只保留新值，避免轮询叠加 DOM 节点。

- 数值变大：新值从上方进入并向下落位，旧值向上离场；入场值带轻微放大；
- 数值变小：新值从下方进入并向上归位，旧值向下离场；入场值带轻微缩小；
- 动画使用 `transform` 和 `opacity`，不使用会触发布局重排的高度/边距动画；
- 指标视口 `overflow: hidden`，更新期间保持高度与列宽稳定，避免 dashboard 抖动。

方向判断使用状态快照中的原始数值，不解析已经格式化的中文字符串。每个指标维护自己的上一次有效数值，状态请求失败或快照过期不会伪造方向。

### 可访问性与性能

动态值通过现有安全的 `textContent` 更新路径写入；不把服务端文本拼接进 HTML。指标的语义标签保持不变，数值更新不会重复制造额外的可读文本。`prefers-reduced-motion: reduce` 下禁用关键帧和过渡，直接显示最新值。动画只使用合成友好的 `transform`/`opacity`，持续轮询时新一轮更新会取消当前指标旧动画并以最新快照为准。

## 实现边界

- 修改范围限定在 `ds4_server.c` 的 dashboard 内嵌 CSS、指标标记和前端状态渲染逻辑；
- 复用现有 `text()` 和 `paintMonitor()` 更新流程，但为数值指标增加原始比较值和方向元数据；
- 不引入前端框架、额外静态资源、服务端接口或新的轮询频率；
- 保留现有主题、响应式断点、进度条、模式切换和 reduced-motion 规则；
- 只替换当前通用的 `.value-changing` 淡入下移动效，避免同一指标同时触发两套动画。

## 测试与验收

在 `tests/dashboard_ui_test.js` 增加指标动效断言：

1. 初始快照不播放方向动效，但六个指标内容正确显示；
2. 将预填充、解码、命中率、上下文利用率和队列值分别调高时，目标指标暴露 `increase` 方向并使用向下入场动画；
3. 将同一组值调低时，目标指标暴露 `decrease` 方向并使用向上入场动画；
4. 值不变、值不可用或请求失败时不新增方向动画；
5. 连续轮询不会累积多个值层，指标高度保持稳定；
6. 开启 reduced-motion 时内容更新仍正确且不运行关键帧；
7. 保持现有 1440px、平板和 390px 无溢出测试，以及 `make ds4_test && ./ds4_test --server` 回归。

视觉验收以 B 方案为准：方向明显、滚动有仪表感，但不造成列宽跳动；监控模式之外不出现新的数字动效。

## 非目标

本次不实现逐位里程表滚动、趋势图、历史数据、声音反馈、管理模式数值动画或表格行内动画。
